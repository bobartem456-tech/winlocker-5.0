​📋 ТЕХНИЧЕСКАЯ ДОКУМЕНТАЦИЯ И ТЗ НА ДОРАБОТКУ: Telegram-бот "Родительский контроль / Удаленное администрирование Windows"
​1. Описание проекта
​Проект представляет собой систему удаленного администрирования и родительского контроля для ПК на базе ОС Windows. Управление осуществляется исключительно через Telegram-бота.
​Главный принцип работы: Бот устанавливается на целевой ПК, работает в скрытом (стелс) режиме, собирает данные, управляет процессами/файлами и отправляет результаты администратору в Telegram.
​2. Текущая Архитектура
​Язык разработки: Python 3.x.
​База данных: SQLite (локальная БД хранит зарегистрированные устройства, список Telegram ID администраторов и логи действий).
​Связь с ОС Windows: Осуществляется через встроенные библиотеки (os, subprocess), а также через кастомную динамическую библиотеку system_core.dll (написана на C/C++) для низкоуровневых системных вызовов (используется через модуль ctypes).
​Модуль выживаемости (Watchdog): Отдельный процесс watchdog.exe, который следит за тем, чтобы основной процесс бота не был завершен, и перезапускает его в случае падения.
​3. Что УЖЕ реализовано и РАБОТАЕТ
​Многие базовые команды уже функционируют корректно.
​Исправлена ошибка 400 (Parse Mode HTML): В текстах команд (например, /coms) все угловые скобки < > заменены на квадратные [ ], чтобы Telegram не пытался читать их как HTML-теги.
​Обход лимитов сообщений: Длинные ответы (например, команда /history для истории браузера или длинные списки) бот успешно формирует в оперативной памяти с помощью io.BytesIO(), присваивает им имя (например, history.txt) и отправляет через метод send_document.
​Текущий рабочий арсенал команд:
​Управление ПК: /pc_list, /admin_list, /uninstall, /rename_bot, /rename_wd, /rename_pc.
​Инфо: /screen (скриншот), /history (отправка .txt файла), /app_list (список установленных программ).
​Питание и процессы: /procs, /kill, /lock, /poweroff, /reboot.
​Система: /volume mute, /message, /cmd (стейт-машина с запросом команды).
​Базовый интерфейс: /panel (выбор ПК), /coms.
​4. 🔴 ЗАДАЧИ ДЛЯ НОВОГО РАЗРАБОТЧИКА (БЕКЛОГ БАГОВ И ФИЧ)
​Код требует рефакторинга. Присутствуют критические логические ошибки и нарушения стелс-режима. Ниже приведен жесткий список задач на исправление:
​Задача 1: Глобальная ошибка Базы Данных (log_action)
​Симптом: При выполнении многих действий (переименование, изменение папки и др.) падает ошибка: Database.log_action() got an unexpected keyword argument 'details'.
​Решение: Изменить сигнатуру метода log_action в классе БД. Сделать её такой: def log_action(self, user_id, action, details=None):. Убедиться, что БД корректно проглатывает или записывает аргумент details.
​Задача 2: Нарушение Стелс-режима (Всплывающие окна)
​Симптом: При выполнении системных команд (особенно /volume, /rename и вызовах CMD) на экране на долю секунды появляются черные окна консоли PowerShell или CMD.
​Решение: Полностью отказаться от os.system. ВЕЗДЕ использовать subprocess.run или subprocess.Popen. На Windows в каждый такой вызов ОБЯЗАТЕЛЬНО передавать флаг скрытия окна: creationflags=subprocess.CREATE_NO_WINDOW (или 0x08000000).
​Задача 3: Разработка Telegram-Проводника (Интерактивная работа с файлами)
​Симптом: Старая система команд (/list_files, /rm, /change_dir) не работает или работает с ошибками (конфликт локальных переменных os).
​Решение: 1. Полностью удалить старые команды файлов из кода и из меню /coms.
2. Перевести работу с файлами в Inline-кнопки внутри /panel (кнопка "📁 Файловый менеджер").
3. Логика проводника:
​Корень: При открытии отображаются логические диски (C:, D:).
​Пагинация: Если в папке сотни файлов, выводить по 10-15 на страницу. Добавить Inline-кнопки [ ⬅️ Назад ] и [ Вперед ➡️ ] для перелистывания страниц.
​Родительская папка: В каждой открытой папке первой кнопкой должна быть [ ⬆️ В предыдущую папку ] (переход на уровень выше через os.path.dirname).
