# АРХИТЕКТУРА ПРОЕКТА WINLOCKER 5.0
# Версия: 16.0
# Дата обновления: 2026-03-22

## ОБЩЕЕ ОПИСАНИЕ
WinLocker 5.0 - это многофункциональный Telegram бот для удаленного администрирования Windows-устройств с поддержкой мульти-администраторов, шифрования данных и стелс-режима.

## СТРУКТУРА ПРОЕКТА

### ОСНОВНЫЕ МОДУЛИ

1. **main_bot.py** - Главный файл запуска
   - Инициализация системы
   - Проверка зависимостей
   - Запуск Telegram бота
   - Управление единичным экземпляром (single instance)
   - Запуск unified logger

2. **bot_core.py** - Ядро системы
   - Инициализация бота и конфигурации
   - Управление ролями (SuperAdmin, Admin)
   - Система состояний (state machine)
   - Функции автозапуска (тройная система)
   - Регистрация устройств
   - Отправка приветственных сообщений

3. **bot_commands.py** - Обработчики команд
   - Все текстовые команды (/start, /panel, /info, /screen и т.д.)
   - Интерактивный ввод (state machine для /cmd)
   - Подтверждение опасных действий
   - Команда /coms (список всех команд)

4. **bot_callbacks.py** - Обработчики callback-запросов
   - Inline-кнопки панели управления
   - Файловый менеджер
   - Подтверждение действий
   - Интерактивные элементы

5. **bot_services.py** - Системные сервисы
   - Скриншоты, веб-камера, микрофон
   - Управление процессами
   - Системные команды
   - Работа с файлами
   - Сканирование системы
   - История браузера

6. **database.py** - Управление базой данных
   - SQLite база данных
   - Таблицы: admins, devices, sessions, action_logs
   - Методы CRUD для администраторов и устройств
   - Очистка дубликатов
   - Логирование действий

7. **unified_logger.py** - Объединенный логгер
   - Мониторинг клавиатуры (pynput)
   - Мониторинг буфера обмена (pyperclip)
   - Локальное шифрование данных
   - Хронологическая запись с таймкодами
   - Обнаружение смены активного окна

8. **system_dll.py** - Системные DLL-функции
   - Интерфейс для системных операций
   - Стелс-режим (CREATE_NO_WINDOW)
   - Управление громкостью
   - Блокировка/разблокировка рабочей станции

9. **readme.py** - Документация и руководство
   - Команда /guide
   - Системная информация
   - Быстрый старт

### ВСПОМОГАТЕЛЬНЫЕ ФАЙЛЫ

10. **config.py** - Конфигурация
    - Токен бота
    - ID SuperAdmin
    - Константы ролей
    - Настройки путей

11. **watchdog.py** - Watchdog процесс
    - Мониторинг основного бота
    - Автоматический перезапуск при сбоях
    - Защита от завершения

12. **build_*.bat** - Скрипты сборки
    - Компиляция в .exe через PyInstaller
    - Создание иконок
    - Упаковка зависимостей

## АРХИТЕКТУРНЫЕ ПРИНЦИПЫ

### 1. МОДУЛЬНОСТЬ
Каждый модуль отвечает за конкретную функциональность, что упрощает поддержку и тестирование.

### 2. БЕЗОПАСНОСТЬ
- Шифрование логов (unified_logger.py)

### 3. НАДЕЖНОСТЬ

### 4. СТЕЛС-РЕЖИМ

- Минимальный след в системе

### 5. МАСШТАБИРУЕМОСТЬ
- Поддержка множества устройств
- Мульти-администраторская модель
- Модульная архитектура для добавления новых функций

## ПОТОКИ ДАННЫХ

### ЗАПУСК СИСТЕМЫ:
1. main_bot.py → проверка single instance
2. Инициализация config.py и database.py
3. Регистрация устройства в БД
4. Настройка автозапуска
5. Запуск unified logger
6. Отправка приветственного ping SuperAdmin
7. Запуск polling Telegram бота

### ОБРАБОТКА КОМАНД:
1. Пользователь отправляет команду
2. Декоратор @command_handler проверяет права
3. Выполняется обработчик из bot_commands.py
4. При необходимости используется state machine
5. Результат отправляется пользователю
6. Действие логируется в БД

### ФАЙЛОВЫЙ МЕНЕДЖЕР:
1. Пользователь нажимает кнопку "Файловый менеджер"
2. Callback обрабатывается в bot_callbacks.py
3. Формируется список файлов с пагинацией
4. Отправляется inline-клавиатура с файлами
5. Пользователь может просматривать, скачивать, удалять файлы

## ТЕХНИЧЕСКИЕ ДЕТАЛИ

### БАЗА ДАННЫХ (SQLite)
Таблицы:
- admins: telegram_id, username, role, created_at
- devices: hwid, device_name, ip_address, last_online, watchdog_status
- sessions: admin_id, device_id, started_at, ended_at
- action_logs: admin_id, device_id, action_type, details, timestamp

### ЛОГИРОВАНИЕ
Два уровня:
1. Unified logger (шифрованные логи клавиатуры и буфера обмена)
2. Action logs в БД (действия администраторов)

### ОБРАБОТКА ОШИБОК
Централизованная система через декоратор @command_handler:
- Перехват исключений
- Логирование с traceback
- Пользовательские сообщения об ошибках

### МНОГОПОТОЧНОСТЬ
- Telegram polling в основном потоке
- Unified logger в отдельных потоках
- Длительные операции (сканирование, загрузка) в потоках

## КЛЮЧЕВЫЕ ФИЧИ ПОСЛЕДНЕЙ ВЕРСИИ (5.0)

1. **Исправление базы данных**: Метод log_action теперь принимает параметр 'details' вместо 'action_details'
2. **Абсолютный стелс**: Все os.system заменены на subprocess.run с CREATE_NO_WINDOW
3. **Умный кейлоггер**: Хронологическая запись с обработкой Backspace как '<', Delete как '>'
4. **Параметры времени**: Команды /mic [секунды] и /webcam [секунды] принимают параметры времени
5. **Обход лимитов Telegram**: /history и /app_list всегда отправляются как файлы
6. **Улучшенный файловый менеджер**: Показ размеров, дат и типов файлов
7. **Интерактивный /cmd**: При вызове без аргументов переходит в интерактивный режим
8. **Восстановленное приветствие**: При запуске отправляется "🟢 ПК [Имя] в сети. IP: [IP-адрес]"

## КОМАНДЫ (/coms)

### ОСНОВНЫЕ
- /start - Приветствие и регистрация
- /panel - Панель управления с inline-кнопками
- /pc_list - Список всех устройств
- /info - Информация о текущем устройстве

### МОНИТОРИНГ
- /screen - Скриншот экрана
- /active - Активное окно/приложение
- /procs - Список процессов
- /keyboard - Лог клавиатуры и буфера обмена
- /webcam [секунды] - Захват с веб-камеры
- /mic [секунды] - Запись с микрофона

### СИСТЕМНЫЕ ОПЕРАЦИИ
- /cmd [команда] - Выполнение команды CMD (интерактивный режим при вызове без аргументов)
- /lock - Блокировка компьютера (требует подтверждения)
- /poweroff - Выключение (требует подтверждения)
- /reboot - Перезагрузка (требует подтверждения)
- /kill [имя_процесса] - Завершение процесса
- /volume [уровень] - Установка громкости (0-100)

### ФАЙЛОВЫЕ ОПЕРАЦИИ
- /list_files [путь] - Просмотр файлов в директории
- /change_dir [путь] - Изменение рабочей директории
- /download [путь] - Загрузка файла с устройства
- /rm [путь] - Удаление файла (требует подтверждения)

### АДМИНИСТРИРОВАНИЕ
- /add_admin [id] - Добавить администратора (только SuperAdmin)
- /del_admin [id] - Удалить администратора (только SuperAdmin)
- /admin_list - Список администраторов
- /cleanup_db - Очистка дубликатов в БД

### ОБНОВЛЕНИЯ
- /urs_update [ссылка] - OTA обновление по ссылке
- /wd_download [url] - Загрузка watchdog.exe
- /wd_on - Запуск watchdog
- /wd_off - Остановка watchdog

### ДОКУМЕНТАЦИЯ
- /guide - Подробный справочник по архитектуре (этот файл)
- /coms - Список всех команд с описаниями
