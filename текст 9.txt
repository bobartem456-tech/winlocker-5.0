6. Тестирование и критерии приёмки

Тестовая среда:

· Windows 10/11 с последними обновлениями.
· Установлены антивирусы: Windows Defender (включён), Kaspersky Free, Dr.Web (демо-версия), ESET NOD32 (триал).
· Изолированная виртуальная машина без выхода в интернет (кроме прокси для бота).

Критерии:

1. Скрытность процессов:
   · В диспетчере задач нет процессов с именами, содержащими bot, watchdog, python и т.п.
   · В Process Hacker нет подозрительных DLL, загруженных в системные процессы, кроме нашей (но её имя должно быть случайным, зашифрованным).
2. Отсутствие файлов на диске:
   · Поиск по диску C: не находит файлов с расширениями .py, .exe, .dll, содержащих строки из кода бота.
   · Альтернативные потоки NTFS не сканируются антивирусом (но мы проверяем, что данные там лежат зашифрованные).
3. Устойчивость:
   · Принудительное завершение основного бота (через Process Hacker) приводит к его автоматическому перезапуску в течение 30 секунд.
   · После перезагрузки системы бот активен (проверяем по наличию пайпа или мьютекса).
4. Обход детекции:
   · Запуск бота и выполнение команд /screen, /history, /procs не вызывает предупреждений антивирусов (проверяем журнал защитника, уведомления Касперского).
5. Сетевой трафик:
   · В Wireshark все запросы к API Telegram выглядят как обычные браузерные (TLS handshake с fingerprint Chrome, User-Agent Chrome). Нет подозрительных паттернов (например, частых коротких соединений).
6. Эмуляция активности:
   · При мониторинге с помощью инструмента Sysmon видна регулярная активность клавиатуры/мыши даже при отсутствии пользователя (но не мешающая реальной работе).

7. Дополнительные рекомендации и предостережения

· Использование polymorphic кода: При каждой установке генерировать уникальные имена мьютексов, пайпов, ключей реестра, чтобы избежать сигнатурных детектов. Это можно сделать скриптом на этапе сборки.
· Избегание частых вызовов API, характерных для вредоносного ПО: например, CreateRemoteThread использовать только при необходимости, предпочитая APC.
· Мониторинг целостности: Если антивирус изменяет наш код в памяти (например, восстанавливает AMSI), watchdog должен периодически повторно патчить.
· Легальность: Данный инструмент предназначен для родительского контроля и администрирования только с согласия владельца ПК. Использование без согласия незаконно.
