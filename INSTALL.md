## 🔧 Установка и запуск

### Вариант 1: Запуск через Python (рекомендуется для тестирования)

#### 1. Установите зависимости
```cmd
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

**Примечание**: Если возникают ошибки установки, попробуйте:
```cmd
python -m pip install --break-system-packages -r requirements.txt
```

#### 2. Настройте конфигурацию
Создайте файл `.env` в корневой папке проекта:

```env
BOT_TOKEN=ваш_токен_telegram_бота
SUPER_ADMIN_ID=ваш_id_в_telegram
```

**Где взять:**
- `BOT_TOKEN`: Получите от @BotFather в Telegram
- `SUPER_ADMIN_ID`: Узнайте от @userinfobot в Telegram

#### 3. Запустите бота
```cmd
python main_bot.py
```

Или с Watchdog (для автоматического перезапуска):
```cmd
python watchdog.py
```

---

### Вариант 2: Компиляция в EXE

#### 1. Установите PyInstaller
```cmd
python -m pip install pyinstaller
```

#### 2. Скомпилируйте проект
Используйте готовый скрипт:
```cmd
build.bat
```

Или вручную:
```cmd
pyinstaller --clean main_bot.spec
pyinstaller --clean watchdog.spec
```

#### 3. Скопируйте конфигурационные файлы в dist/
```cmd
copy config.py dist\
copy secure_config.py dist\
copy .env dist\
```

#### 4. Запустите из dist/
```cmd
cd dist
MainBot.exe
```

Или с Watchdog:
```cmd
Watchdog.exe
```

## 🐛 Решение проблем

### Ошибка: "No module named 'telebot'"
**Решение**: Установите зависимости:
```cmd
python -m pip install pyTelegramBotAPI
```

### Ошибка: "BOT_TOKEN не настроен"
**Решение**: Создайте файл `.env` с правильным токеном

### Ошибка: "SUPER_ADMIN_ID не настроен"
**Решение**: Добавьте SUPER_ADMIN_ID в `.env` файл

### Ошибка компиляции PyInstaller
**Решение**: 
1. Обновите PyInstaller: `python -m pip install --upgrade pyinstaller`
2. Убедитесь, что все зависимости установлены
3. Попробуйте очистить кэш: `pyinstaller --clean main_bot.spec`

### Бот не запускается после компиляции
**Решение**: 
1. Убедитесь, что `.env` файл находится рядом с .exe
2. Проверьте, что токен и ID правильные
3. Запустите с правами администратора

## 🔐 Безопасность

### ⚠️ ВАЖНО!
1. **НИКОГДА** не коммитьте `.env` файл в репозиторий
2. **НИКОГДА** не храните токены в коде
3. Используйте `.gitignore` для защиты данных

### Лучшие практики:
- Храните `.env` файл в надежном месте
- Используйте переменные окружения для production
- Регулярно меняйте токены бота
- Ограничьте доступ к файлам бота

## 📞 Поддержка

При возникновении проблем:
1. Проверьте логи в `bot_runtime.log`
2. Убедитесь, что все зависимости установлены
3. Проверьте конфигурацию в `.env`
4. Убедитесь, что у бота есть права доступа
